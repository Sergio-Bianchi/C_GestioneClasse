32   <a name="l11"><span class="ln">11   </span></a><span class="s0">## </span><span class="s2">Write content</span>
33   <a name="l12"><span class="ln">12   </span></a><span class="s2">%product% supports two types of markup</span><span class="s1">: </span><span class="s2">Markdown and XML.</span>
34   <a name="l13"><span class="ln">13   </span></a><span class="s2">When you create a new help article, you can choose between two topic types, but this doesn't mean you have to stick to a single format.</span>
35   <a name="l14"><span class="ln">14   </span></a><span class="s2">You can author content in Markdown and extend it with semantic attributes or inject entire XML elements.</span>
36   <a name="l15"><span class="ln">15   </span></a>

65   <a name="l44"><span class="ln">44   </span></a><span class="s0">### </span><span class="s2">Collapsible blocks</span>
66   <a name="l45"><span class="ln">45   </span></a><span class="s2">Apart from injecting entire XML elements, you can use attributes to configure the behavior of certain elements.</span>
67   <a name="l46"><span class="ln">46   </span></a><span class="s2">For example, you can collapse a chapter that contains non-essential information</span><span class="s1">:</span>
68   <a name="l47"><span class="ln">47   </span></a>
69   <a name="l48"><span class="ln">48   </span></a><span class="s0">#### </span><span class="s2">Supplementary info {collapsible=</span><span class="s1">&quot;</span><span class="s2">true</span><span class="s1">&quot;</span><span class="s2">}</span>
70   <a name="l49"><span class="ln">49   </span></a><span class="s2">Content under a collapsible header will be collapsed by default,</span>
71   <a name="l50"><span class="ln">50   </span></a><span class="s2">but you can modify the behavior by adding the following attribute</span><span class="s1">:</span>
72   <a name="l51"><span class="ln">51   </span></a><span class="s2">`default-state=</span><span class="s1">&quot;</span><span class="s2">expanded</span><span class="s1">&quot;</span><span class="s2">`</span>
